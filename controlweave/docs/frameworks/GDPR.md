# 🇪🇺 GDPR Guide

Using ControlWeave for General Data Protection Regulation (GDPR) compliance management.

## Overview

The GDPR is the EU's comprehensive data protection regulation that applies to organizations processing personal data of EU residents, regardless of where the organization is based. ControlWeave maps GDPR articles to actionable controls.

## Key GDPR Principles

| Principle | Article | Description |
|-----------|---------|-------------|
| Lawfulness, fairness, transparency | Art. 5(1)(a) | Legal basis for processing |
| Purpose limitation | Art. 5(1)(b) | Data used only for stated purposes |
| Data minimization | Art. 5(1)(c) | Collect only necessary data |
| Accuracy | Art. 5(1)(d) | Keep data accurate and up-to-date |
| Storage limitation | Art. 5(1)(e) | Retain only as long as necessary |
| Integrity and confidentiality | Art. 5(1)(f) | Secure processing |
| Accountability | Art. 5(2) | Demonstrate compliance |

## Data Subject Rights

ControlWeave helps track and manage data subject rights requests:
- **Right of Access** (Art. 15) — Subject Access Requests (SARs)
- **Right to Rectification** (Art. 16)
- **Right to Erasure** (Art. 17) — "Right to be forgotten"
- **Right to Restriction** (Art. 18)
- **Right to Data Portability** (Art. 20)
- **Right to Object** (Art. 21)

## Using GDPR in ControlWeave

### Activate the Framework
1. Go to **Frameworks** → **Add Framework**
2. Select **GDPR**
3. Click **Activate**

### Data Governance Integration

Use ControlWeave's Data Governance module for GDPR compliance:
- **Data Inventory**: Map personal data across systems
- **Retention Policies**: Configure and track data retention schedules
- **Data Sovereignty**: Track where data is stored geographically

### Records of Processing Activities (ROPA)

Maintain ROPA documentation in ControlWeave:
1. Go to **Data Governance** → **Processing Activities**
2. Document each processing activity with legal basis, data categories, and recipients
3. Export as PDF for regulatory submissions
