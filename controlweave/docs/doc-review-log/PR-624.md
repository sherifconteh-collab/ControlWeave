## Fixed (real, verified discrepancies)

- **`docs/guides/EVIDENCE.md`, Step 10.1** — the source-type table and its
  lead-in sentence stated "**Only Splunk** performs full live data
  retrieval" and described GitHub as merely creating a
  "configuration-only" evidence record (repository, event types documented,
  no live data). That was true before this PR; `githubService.js` now
  performs real live retrieval (code scanning alerts, Dependabot alerts,
  org audit log, pull requests) exactly like Splunk. Updated the summary
  sentence, the GitHub row, and the tip callout to say "Splunk and GitHub."
- **`docs/guides/EVIDENCE.md`, Step 10.2** — the rule-creation steps only
  documented Splunk's config fields (Search Query / Earliest Time / Latest
  Time / Max Events). Added the equivalent GitHub fields (Repository /
  Event Type / Time Range / Max Results), which are now real, working
  inputs as of this PR (previously the rule-creation form silently
  submitted `source_config: {}` for every source type, GitHub included).
- **`docs/guides/EVIDENCE.md`, Step 10.4** — verified against
  `frontend/src/app/dashboard/evidence/page.tsx`: there is a
  standalone "Import from Splunk" one-time-import button on the Evidence
  page, but no equivalent one-time GitHub import button — `importGithubEvidence`
  exists in `lib/api.ts` but isn't wired to any Evidence-page UI. Initially
  drafted a claim that an equivalent GitHub button exists; caught it against
  the actual frontend code before finalizing and corrected it to a note
  that the one-time-import workaround for GitHub today is a `Manual only`
  Auto-Evidence rule run on demand (10.3), not a dedicated button.
- **`docs/guides/SETTINGS.md`, Integrations section** — had a Splunk
  configuration subsection but no GitHub subsection at all, even though
  this PR adds a full "GitHub Evidence Connector" card to
  **Settings → Integrations** (token field, Test Connection, Save,
  Remove — verified against `dashboard/settings/page.tsx` lines
  3005-3057). Added a "GitHub Evidence Connector" subsection describing
  the real fields and actions.
