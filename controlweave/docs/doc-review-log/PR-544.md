Performed live by a Claude Code session with direct PR/repo access (no
GitHub Action, no separate API key) — see the session's PR comment for
context on why this replaced `claude-doc-review.yml`.

Scope: the doc guides `docs-pipeline.yml`'s automated feature-change
detector flagged as affected by this PR's most recent commits
(authentication, frameworks, controls, ai-copilot, ai-analysis,
auditor-workspace, reports, settings), reviewed against the actual code
changes in this PR's auth/RBAC/auditor-workspace/AI-feature fix batches.

## Fixed (real, verified discrepancies)

- **`docs/guides/GETTING_STARTED.md`** — registration password requirement
  said "min 12 characters"; the actual enforced minimum
  (`utils/passwordPolicy.js` `MIN_PASSWORD_LENGTH`) is **15**, and as of
  this PR that complexity check (uppercase/lowercase/digit/special
  character) is now enforced on registration too, not just invite
  acceptance. Updated to state the real requirement.
- **`docs/guides/ACCOUNT_SETUP.md`** — same "12 characters" error in the
  Password Management section's requirement list (which also said
  "Special characters recommended" — it's mandatory, not recommended) and
  in the account-setup checklist's "12+ characters" line. All three fixed.

## Verified already correct — no change needed

- **`docs/guides/AUDITOR_WORKSPACE.md`** — already documented the public
  workspace link showing "Engagement name, type, and status." The code
  didn't actually render the name (interface didn't even include the
  field) until this PR's auditor-workspace fix — the doc was right, the
  app is now compliant with it.
- **`docs/guides/AI_ANALYSIS.md`** — already has a "No LLM Provider
  Configured" troubleshooting entry describing the exact behavior this
  PR restores for the Regulatory Impact and Remediation Plan features
  (previously those two specific features silently returned a generic
  500 instead of the documented error, so the troubleshooting entry
  didn't actually apply to them until now).
- `docs/guides/AI_COPILOT.md`, `docs/guides/CONTROLS.md`,
  `docs/guides/FRAMEWORKS.md`, `docs/guides/REPORTS.md`,
  `docs/guides/SETTINGS.md`, `docs/USER_GUIDE.md` — spot-checked against
  this PR's diff; none of this batch's changes (AI task-profile/schema

  anything these pages describe to a user. No edits made.

## Noted, not fixed (out of scope for this PR)

- **`docs/TIER_COMPARISON.md`** gates "Auditor Workspace" and "External
  Auditor Role" behind a paid tier ("No" on the free tier). Per
  `.claude/rules/tier-system.md`, all tier gating was removed repo-wide —
  `requireTier()`/`requireProEdition()` are no-ops and every feature is
  available to every authenticated user, confirmed for this file's own
  route (`auditorWorkspace.js` has no `requireTier` call). This
  contradiction predates this PR and spans the whole document, not just
  the rows this PR's changes touch — flagging for a dedicated cleanup
  pass rather than partially rewriting a tier-comparison doc outside this
  PR's actual scope.
- **`docs/guides/USERS.md`** — describes role/permission management only
  generically ("assign roles, and manage permissions"). This PR added a
  real restriction (a `roles.manage`/`users.manage` holder can no longer
  grant a permission or role they don't already hold themselves) that
  changes what admins can do in the Roles UI. Worth a sentence, but this
  file wasn't flagged by the automated feature-change detector for this
  diff and isn't in this PR's stated scope — noting rather than editing
  unprompted.
