# Documentation review — PR #581

Performed live by the Claude Code session subscribed to this PR, per
`.claude/rules/doc-review.md`, triggered by `docs-pipeline.yml`'s
"📚 Documentation Update Queued" comment (flagged `controls`: ui-change,
affecting `docs/guides/CONTROLS.md`, `docs/guides/GETTING_STARTED.md`,
`docs/USER_GUIDE.md`; the automated generator itself produced no diff,

Scope: fix for a user-reported bug — control test-result changes were never
recorded to `audit_logs` (so the "Control Testing" card's Save button
silently destroyed the previous verdict with no trace), and the one existing
history widget ("Status History Timeline") mislabeled any test-result audit
entries that did reach it as "Not Started → Not Started" by applying the
wrong status vocabulary. Fix adds a `test_result_changed` audit log entry and
a new, correctly-labeled "Test Result History" timeline on the control detail
page (`controlweave/frontend/src/app/dashboard/controls/[id]/page.tsx`).
