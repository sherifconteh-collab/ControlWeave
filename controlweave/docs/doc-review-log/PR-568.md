## Fixed (real, verified discrepancies)

- **`docs/guides/REPORTS.md`** — had no mention of scheduled report
  delivery at all, even though the feature (recurring PDF/CSV/JSON
  reports emailed to configured recipients) has had a full UI and DB
  schema since before this PR. It was undocumented *and* non-functional
  before this PR (see the PR description); now that
  `runScheduledReport()` actually generates and emails a file and
  `reportScheduler.js` actually triggers it on schedule, added a
  "Scheduled Report Delivery" section describing the 5 report types, 4
  cadences, 3 formats, and the automatic-delivery behavior (including
  what happens when no SMTP transport is configured — the run still
  records but delivery is silently skipped, which is worth calling out
  rather than leaving users to wonder why no email arrived).
- **`README.md`** (root) — already fixed as part of this PR's own diff,
  not a separate doc-review pass: the Android "full feature parity"
  overclaim, the MCP tool count (51 → 54), and four dead links to MCP
  docs `PR #544` deleted (`MCP_TOOLS_REFERENCE.md`,
  `MCP_SECURITY_GUIDE.md`, `CE_MCP_SECURITY_GUIDE.md`,
  `MAESTRO_CEMCP_GUIDANCE.md`, now pointing at `MCP_GUIDE.md` /
  `CE_MCP_GUIDE.md`). Listed here for completeness since the pipeline's
  feature-change detector flagged `readme` for this PR.
- **`controlweave/docs/MCP_GUIDE.md`** — not flagged by the pipeline
  (only root `README.md`/`controlweave/README.md` were), but discovered
  while fixing the root README's tool count: this file repeated the same
  stale "51 tools" figure in two places, plus a per-category breakdown
  that summed to 46, not 51 or 54. Recounted from
  `mcp-tool-registry.js`'s actual `category:` fields (54 total, exact
  per-category split) and corrected both the headline number and the
  breakdown table.
