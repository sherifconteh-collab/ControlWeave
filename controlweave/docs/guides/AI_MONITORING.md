# 🔍 AI Monitoring Guide

Define monitoring rules for your AI systems and track policy violations, threshold breaches, and anomalous behavior in real time.

## Overview

The AI Monitoring module allows you to create rules that watch for specific conditions in your AI agents and systems — such as metric thresholds, output patterns, anomalies, or policy violations — and generate events when those conditions are triggered.

---

## Monitoring Dashboard

Navigate to **AI Monitoring** to see your monitoring summary:

- **Total AI Systems** — AI agents registered in your CMDB
- **Monitored Systems** — AI agents with at least one active rule
- **Active Rules** — total enabled monitoring rules
- **Events Last 24h** — monitoring events triggered in the last 24 hours
- **Open Events** — events that have not been resolved
- **Critical Open Events** — unresolved events with critical severity
- **High Open Events** — unresolved events with high severity
- **Last Event Time** — timestamp of the most recent event

## Monitoring Rules

Rules define the conditions that trigger monitoring events.

### Rule Types

| Type | Description |
|------|-------------|
| **Threshold** | Triggers when a metric crosses a defined value (e.g., latency > 2s) |
| **Pattern** | Triggers when output matches a regular expression pattern |
| **Anomaly** | Triggers when behavior deviates from established baseline |
| **Policy Violation** | Triggers when output or behavior violates a defined policy |

### Creating a Monitoring Rule

1. Go to **AI Monitoring** → **Rules**
2. Click **Create Rule**
3. Fill in the rule configuration:
   - **Rule Name** — descriptive name (e.g., "High Latency Alert")
   - **Rule Type** — Threshold, Pattern, Anomaly, or Policy Violation
   - **Description** — optional explanation of what this rule monitors
   - **AI Agent** (optional) — link to a specific AI agent asset from your CMDB
   - **Framework Controls** (optional) — link to relevant compliance controls
4. Configure type-specific settings:
   - **Threshold rules**: Metric name, threshold value, operator (>, <, =, ≥, ≤)
   - **Pattern rules**: Regular expression to match against AI outputs
   - **Evaluation Window** — time window in seconds (default: 300)
5. Set alert configuration:
   - **Alert Severity** — Low, Medium, High, Critical (default: Medium)
   - **Block on Violation** — whether to block the AI action when rule triggers
   - **Require Human Review** — whether a human must review before proceeding
   - **Notification Targets** — who/what to notify when rule triggers
6. Click **Create Rule**

### Managing Rules

- **Enable/Disable** — toggle rules on/off without deleting them
- **Edit** — update rule configuration, thresholds, or notification targets
- **Delete** — permanently remove a rule

## Monitoring Events

Events are generated when a monitoring rule is triggered.

### Event Severities

- **Critical** — immediate action required; may involve blocking behavior
- **High** — significant issue needing prompt attention
- **Medium** — notable anomaly for review
- **Low** — informational, low urgency

### Event Statuses

- **Open** — event triggered, not yet resolved
- **Acknowledged** — someone has seen the event
- **Resolved** — event has been addressed

### Reviewing Events

1. Go to **AI Monitoring** → **Events**
2. Filter by severity, status, or AI agent
3. Click an event to see:
   - Which rule triggered it
   - Which AI agent was involved
   - Event details and context
   - Whether human review is required
4. Update the event status as you address it

## Linking Rules to Compliance Controls

You can associate monitoring rules with compliance framework controls to demonstrate continuous monitoring:

1. When creating or editing a rule, add **Framework Control IDs**
2. The monitoring activity is visible in the associated control's detail view
3. This supports controls like NIST 800-53 CA-7 (Continuous Monitoring) and SI-4 (System Monitoring)

> **💡 Tip**: Linking monitoring rules to controls provides automated evidence of continuous monitoring, which is valuable during audits.

---

## AI Agent Integration

Monitoring rules can target specific AI agents registered in your CMDB:

1. Go to **CMDB** → add an asset with the **AI Agent** category
2. When creating a monitoring rule, select that asset from the **AI Agent** dropdown
3. Events generated by the rule will reference that specific AI agent
