# 📋 Audit Engagements Guide

Manage audit engagements within the ControlWeave Auditor Workspace.

## Overview

Audit engagements represent formal audit projects between your organization and an external auditor. Each engagement tracks scope, timeline, team members, and associated workpapers and findings.

## Creating an Engagement

1. Navigate to **Auditor Workspace** → **Engagements**
2. Click **New Engagement**
3. Fill in engagement details:
   - **Name**: e.g., "SOC 2 Type II — Q1 2026"
   - **Framework**: Select the applicable compliance framework
   - **Start Date / End Date**: Engagement timeline
   - **Lead Auditor**: Assign the primary auditor
4. Click **Create**

## Managing Engagement Status

Engagements move through the following lifecycle:

| Status | Description |
|--------|-------------|
| **Planning** | Engagement created, scope being defined |
| **Fieldwork** | Active testing and evidence review |
| **Review** | Draft findings being reviewed |
| **Reporting** | Final report preparation |
| **Closed** | Engagement complete |

## Engagement Scope

Define what systems, controls, and frameworks are in scope:

1. Open the engagement
2. Click **Edit Scope**
3. Select frameworks, control families, or specific controls
4. Save scope definition

## Team Management

Add auditors and organization members to the engagement:

1. Open the engagement → **Team** tab
2. Click **Add Member**
3. Select role: **Lead Auditor**, **Auditor**, or **Auditee**
4. Send invitation if the user doesn't have an account
