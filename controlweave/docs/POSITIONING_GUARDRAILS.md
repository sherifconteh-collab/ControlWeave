# Positioning Guardrails

Guidelines for consistent messaging and positioning of the ControlWeave platform across all channels.

## Core Message

**"Audit the code that audits your AI — then let us run it for you."**

## Guardrails

### Do

- Lead with auditability and transparency
- Emphasize AGPL v3 open-source credibility
- Reference specific frameworks (NIST AI RMF, EU AI Act Article 17, ISO 42001)
- Differentiate on federal GRC pedigree

### Don't

- Claim features that are tier-gated without noting the tier requirement
- Compare directly to competitors by name in public content
- Promise FedRAMP authorization (say "FedRAMP-ready architecture")
- Use "free forever" language — use "Community Edition (AGPL v3, self-hosted)"

## Tier Messaging

| Tier | Message Focus |
|------|--------------|
| Community | Open-source, self-hosted, full audit rights |
| Pro | Managed SaaS, SSO, standard support |
| Enterprise | BYOK, RBAC, dedicated CSM, evidence packs |
| Gov Cloud | FedRAMP-ready, IL4/IL5, ITAR-compliant |
